\doxysubsection{Detailed Description}
Manages the list of geographic maps. 

This class manages a list of remotely available and locally installed databases and geographic maps. More specifically, it manages the following items.


\begin{DoxyItemize}
\item Aviation maps (in Geo\+JSON format, file name ends in "{}geojson"{})
\item Base maps/raster (in MBTILES format, file name ends in "{}mbtiles"{})
\item Base maps/vector (in MBTILES format, file name ends in "{}raster"{})
\item Terrain maps (in MBTILES format, file name ends in "{}terrain"{})
\item FLARM Databases (as a text file, file name ends in "{}data"{})
\end{DoxyItemize}

In addition, it allows access to the following data.


\begin{DoxyItemize}
\item "{}\+What\textquotesingle{}s new"{}-\/message from remove server
\end{DoxyItemize}

The class retrieves the list of available items from a remote server on a regular basis, updates the list automatically once a week, and maintains a list of Downloadable objects that corresponds to the remotely available and locally installed items.

The address of the remote server is hardcoded into the binary. The list is downloaded to a file "{}maps.\+json"{} in QStandard\+Paths\+::writable\+Location(\+QStandard\+Paths\+::\+App\+Data\+Location). The format of the file is described at this URL\+:

\href{https://github.com/Akaflieg-Freiburg/enrouteServer/wiki/The-file-maps.json}{\texttt{ https\+://github.\+com/\+Akaflieg-\/\+Freiburg/enroute\+Server/wiki/\+The-\/file-\/maps.\+json}}

Locally installed items are saved in the directory "{}aviation\+\_\+maps"{} in QStandard\+Paths\+::writable\+Location(\+QStandard\+Paths\+::\+App\+Data\+Location), or into a suitable subdirectory of this.

The implementation assumes that no other program interferes with the file "{}maps.\+json"{} and with the directory "{}aviation\+\_\+maps"{}.
